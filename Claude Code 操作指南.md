## 📂 路径规则

### Vault 内部（相对路径）

✅ **正确方式**：
```
notes/my-note.md          # 相对于 vault 根目录
daily-notes/2024-01-15.md
../template.md            # 相对路径
```

❌ **错误方式**：
```
/notes/my-note.md         # 不要以 / 开头
/Users/.../file.md        # 不要使用绝对路径
```

### 导出路径（绝对路径或 ~）

只能导出到指定位置：
```
~/Desktop/output.pdf
~/Downloads/backup.md
```

## ⚠️ 注意事项

### 数据安全

1. **备份重要数据**：批量操作前建议备份
2. **确认操作**：Claude 会说明将要做的更改
3. **使用 Git**：对重要笔记启用版本控制

### 性能优化

1. **大型 vault**：搜索和索引可能需要时间
2. **批量操作**：分批处理大量文件
3. **附件处理**：图片、PDF 等大文件需要时间读取

### 最佳实践

1. **明确的指令**：具体说明你想要什么
2. **渐进式操作**：复杂任务分步骤进行
3. **反馈修正**：如果结果不符合预期，告诉 Claude 如何调整

## ❓ 常见问题

**Q: Claude 能访问我的所有笔记吗？**
A: 是的，但只在你明确请求时。Claude 不会自动读取所有内容。

**Q: 我可以撤销 Claude 的更改吗？**
A: 使用 Git 或者让 Claude 恢复之前的版本。建议重要操作前提交。

**Q: Claude 理解 Obsidian 的特殊语法吗？**
A: 是的！包括 Wiki 链接、标签、嵌入、Callouts、Frontmatter 等。
