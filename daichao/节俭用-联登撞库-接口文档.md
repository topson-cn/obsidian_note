# 一、接入前提须知

## 1、流程

- 我方调用合作方，接口由我方提供。

## 2、请求方式

- 请求方式统一为：**POST**
- 内容格式（Content-Type）为：**application/json**

## 3、公共请求头

| Header | 说明 |
|--------|------|
| appUserId | 应用用户ID |
| jiejianyongAppToken | 节俭用App Token |
| jiejianyongToken | 节俭用Token |
| channelNo | 渠道编号 |
| packageKey | 包密钥 |

## 4、公共请求参数

| 参数名称 | 类型 | 是否必填 | 说明 |
|---------|------|---------|------|
| channelCode | String | 是 | 渠道唯一标识，双方对接时提供 |
| data | String | 是 | 参数进行整体 AES 加密后的加密参数内容 |
| reqId | String | 是 | 请求唯一ID |

## 5、公共响应参数

| 参数名称 | 类型 | 是否必填 | 说明 |
|---------|------|---------|------|
| code | Integer | 是 | 响应码 1000 成功，其它失败 |
| msg | String | 是 | 响应说明 |
| data | String | 否 | 响应数据 |

## 6、加密规则说明

- 加密方式：AES ECB, Base64
- 补码方式：PKCS5Padding
- 加密数据：`data = Base64.encode(AES.ECB.encrypt(content, PKCS5Padding, key))`
- 测试密钥：联调时提供
- 正式密钥：上线时提供

# 二、产品掩码8位撞库联登文档（标准）

## 接口 1：小联登撞库

- **接口名称**：1.小联登撞库
- **请求方式**：POST
- **接口地址**：/接入方提供
- **接口ID**：422140027

### 说明

> 注：phoneList 返回参数解释
> 掩码手机号：`1530819000` → 返回结果 `[md5('15308190001'), md5('15308190002'), ...]`
>
> 注：撞库成功判断
> 1. code 等于 1000，phoneList 字段为空
> 2. code 等于 1000，phoneList 返回集合中未包含撞库手机号MD5（撞库手机号完整号码MD5）

### 请求参数（data 解密后的 JSON）

| 参数名称 | 类型 | 是否必填 | 说明 |
|---------|------|---------|------|
| userPhone | String | 是 | 手机号前8位明文 |
| channeCode | String | 是 | 上游渠道编码 |

**请求示例**：
```json
{
    "userPhone": "14512345"
}
```

### 响应参数

| 参数名称 | 类型 | 说明 |
|---------|------|------|
| code | Integer | 接口状态码 1000：成功，其它：失败 |
| msg | String | 响应消息 |
| data | Object | 数据（code为1000时必返） |
| data.phoneList | String[] | 撞库匹配手机号MD5集合数据 |
| data.applyId | String | 进件单号 |

**响应示例**：
```json
{
    "code": 1000,
    "msg": "撞库成功",
    "data": {
        "phoneList": [
            "0ff4485dd39c24da914bbd1b962e61c1"
        ],
        "applyId": "123456789"
    }
}
```

# 三、产品 MD5 撞库联登文档（标准）

## 接口 1：小联登撞库

- **接口名称**：1.小联登撞库
- **请求方式**：POST
- **接口地址**：/接入方提供
- **接口ID**：435999260

### 说明

> 注：身份证 MD5 与手机号 MD5 不能同时为空
>
> 注：撞库成功判断
> 1. code 等于 1000，checkResult 为 true

### 请求参数（data 解密后的 JSON）

| 参数名称 | 类型 | 是否必填 | 说明 |
|---------|------|---------|------|
| phoneNumberMd5 | String | 否* | 手机号MD5（与 idCardMd5 不能同时为空） |
| idCardMd5 | String | 否* | 身份证MD5（与 phoneNumberMd5 不能同时为空） |
| channeCode | String | 是 | 上游渠道编码 |

### 响应参数

| 参数名称 | 类型 | 说明 |
|---------|------|------|
| code | Integer | 接口状态码 1000：成功，其它：失败 |
| msg | String | 响应消息 |
| data | Object | 数据（code为1000时必返） |
| data.checkResult | String | true 通过，false 不通过 |

**响应示例**：
```json
{
    "code": 1000,
    "msg": "撞库成功",
    "data": {
        "checkResult": true
    }
}
```

# 四、接口总览

| 模式 | 接口 | 撞库字段 | 注册字段 | 撞库成功判断 |
|------|------|---------|---------|-------------|
| 掩码8位 | 撞库 | userPhone(前8位) | - | phoneList为空 或 不含完整手机号MD5 |
| 掩码8位 | 注册 | - | applyId + userPhone(完整) | code=1000 返回url |
| MD5 | 撞库 | phoneNumberMd5 / idCardMd5 | - | checkResult=true |
| MD5 | 注册 | - | phoneNumber + idCard(明文) | code=1000 返回url |
