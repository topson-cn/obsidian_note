# 还款前置校验系统 - 业务流

## 概述

还款前置校验系统的业务流涵盖用户还款前的各种校验和审核流程，确保还款操作的安全性和合规性。

## 主要业务流程

### 1. 还款校验流程

#### 流程说明

用户发起还款请求后，系统进行一系列校验，包括灰度校验、规则评估、审计策略评估等。

#### 流程图

```mermaid
graph TD
    Start[用户发起还款] --> ReqCheck[请求参数校验]
    ReqCheck -->|校验失败| ReturnError[返回错误]
    ReqCheck -->|校验成功| GrayCheck[灰度校验]
    GrayCheck -->|在灰度名单| AuditCheck[审计策略校验]
    GrayCheck -->|不在灰度名单| RuleCheck[还款规则校验]

    AuditCheck -->|需要审核| NeedAudit[需要审核流程]
    AuditCheck -->|不需要审核| RuleCheck

    NeedAudit --> SubmitAudit[提交审核]
    SubmitAudit --> AuditResult{审核结果}
    AuditResult -->|通过| RuleCheck
    AuditResult -->|拒绝| ReturnError

    RuleCheck -->|校验失败| ReturnError
    RuleCheck -->|校验成功| ExternalCheck[外部服务校验]
    ExternalCheck -->|校验失败| ReturnError
    ExternalCheck -->|校验成功| Success[允许还款]

    style Start fill:#e1f5ff
    style ReqCheck fill:#fff9c4
    style GrayCheck fill:#fce4ec
    style AuditCheck fill:#f3e5f5
    style NeedAudit fill:#ffe0b2
    style SubmitAudit fill:#fff4e1
    style AuditResult fill:#fff9c4
    style RuleCheck fill:#e8f5e9
    style ExternalCheck fill:#e8f5e9
    style Success fill:#c8e6c9
    style ReturnError fill:#ffcdd2
```

#### 关键节点说明

| 节点 | 说明 | 涉及接口 |
|------|------|---------|
| 请求参数校验 | 校验请求参数的完整性和合法性 | 所有接口 |
| 灰度校验 | 检查用户是否在灰度名单中 | /repayOrder/gray |
| 审计策略校验 | 根据审计策略判断是否需要审核 | /repayAudit/preSubmit |
| 还款规则校验 | 校验还款金额、日期、订单状态等 | /repaySubmit/check |
| 外部服务校验 | 调用外部服务进行二次校验 | 外部 Feign 客户端 |

### 2. 还款审核流程

#### 流程说明

对于触发了审计策略的还款请求，需要走审核流程。审核流程包括预审核、提交审核、审核结果查询等。

#### 流程图

```mermaid
graph TD
    Start[还款请求触发审计] --> PreSubmit[预提交审核]
    PreSubmit --> QueryPolicy[查询审计策略]
    QueryPolicy --> Evaluate[评估策略]
    Evaluate --> CheckWhiteList{在白名单?}
    CheckWhiteList -->|是| Pass[直接通过]
    CheckWhiteList -->|否| CheckLimit{是否超出限额?}
    CheckLimit -->|是| ExceedLimit[超出限额]
    CheckLimit -->|否| NeedAudit[需要人工审核]

    ExceedLimit --> Record[记录审计信息]
    NeedAudit --> Record

    Record --> ReturnPre[返回预提交结果]

    ReturnPre --> IsNeedAudit{需要审核?}
    IsNeedAudit -->|是| SubmitAudit[提交审核]
    IsNeedAudit -->|否| Pass

    SubmitAudit --> Approve[审核人审批]
    Approve --> ApproveResult{审批结果}
    ApproveResult -->|通过| UpdateStatus[更新状态为通过]
    ApproveResult -->|拒绝| UpdateReject[更新状态为拒绝]

    UpdateStatus --> QueryResult[查询审核结果]
    UpdateReject --> QueryResult

    QueryResult --> Notify[通知还款系统]
    Notify --> End[流程结束]

    style Start fill:#e1f5ff
    style PreSubmit fill:#fff4e1
    style QueryPolicy fill:#e8f5e9
    style Evaluate fill:#f3e5f5
    style CheckWhiteList fill:#fce4ec
    style CheckLimit fill:#fff9c4
    style ExceedLimit fill:#ffcdd2
    style NeedAudit fill:#ffe0b2
    style Record fill:#e8f5e9
    style ReturnPre fill:#fff4e1
    style IsNeedAudit fill:#fff9c4
    style SubmitAudit fill:#fff4e1
    style Approve fill:#f3e5f5
    style ApproveResult fill:#fff9c4
    style UpdateStatus fill:#c8e6c9
    style UpdateReject fill:#ffcdd2
    style QueryResult fill:#e8f5e9
    style Notify fill:#fff9c4
    style End fill:#cfd8dc
    style Pass fill:#c8e6c9
```

#### 关键节点说明

| 节点 | 说明 | 涉及接口 |
|------|------|---------|
| 预提交审核 | 评估审计策略，判断是否需要审核 | /repayAudit/preSubmit |
| 提交审核 | 提交到审核队列，等待审核人处理 | /repayAudit/submit |
| 审核人审批 | 审核人登录系统进行审批 | 审核管理系统 |
| 查询审核结果 | 查询审核状态和结果 | /repayAudit/result |

### 3. 灰度发布流程

#### 流程说明

新功能或新规则上线时，通过灰度发布逐步推广，降低风险。

#### 流程图

```mermaid
graph TD
    Start[新功能上线] --> ConfigGray[配置灰度规则]
    ConfigGray --> SmallGray[小范围灰度<br/>1% 用户]
    SmallGray --> Monitor1[监控指标]
    Monitor1 --> Check1{指标正常?}
    Check1 -->|否| Rollback1[回滚]
    Check1 -->|是| ExpandGray1[扩大灰度范围<br/>10% 用户]

    ExpandGray1 --> Monitor2[监控指标]
    Monitor2 --> Check2{指标正常?}
    Check2 -->|否| Rollback1
    Check2 -->|是| ExpandGray2[扩大灰度范围<br/>50% 用户]

    ExpandGray2 --> Monitor3[监控指标]
    Monitor3 --> Check3{指标正常?}
    Check3 -->|否| Rollback1
    Check3 -->|是| FullRelease[全量发布]

    Rollback1 --> Fix[修复问题]
    Fix --> Start

    style Start fill:#e1f5ff
    style ConfigGray fill:#fff4e1
    style SmallGray fill:#fff9c4
    style ExpandGray1 fill:#fff9c4
    style ExpandGray2 fill:#fff9c4
    style Monitor1 fill:#e8f5e9
    style Monitor2 fill:#e8f5e9
    style Monitor3 fill:#e8f5e9
    style Check1 fill:#fff9c4
    style Check2 fill:#fff9c4
    style Check3 fill:#fff9c4
    style Rollback1 fill:#ffcdd2
    style Fix fill:#fff4e1
    style FullRelease fill:#c8e6c9
```

#### 灰度监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| 接口成功率 | 接口调用成功比例 | < 99% |
| 接口响应时间 | 接口平均响应时间 | > 500ms |
| 异常率 | 接口调用异常比例 | > 1% |
| 审核通过率 | 审核通过的比例 | 变化超过 10% |

## 核心业务规则

### 1. 还款规则

- **单期结清**：只含最后一期的逾期结清
- **多期结清**：包含多期全部逾期
- **部分还款**：支持修改金额后提交
- **提前还款**：逾期前的部分还款

### 2. 审计规则

- **优先级**：高优先级策略优先执行
- **白名单**：白名单用户可跳过部分审核
- **限额控制**：超过限额需要人工审核
- **状态控制**：只有状态为 ENABLE 的策略生效

### 3. 灰度规则

- **灰度比例**：按配置的比例分配灰度用户
- **白名单**：白名单用户优先进入灰度
- **标签匹配**：根据用户标签进行灰度匹配
