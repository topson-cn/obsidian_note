# 预审规则配置化设计说明

## 一、设计背景

### 1.1 现状分析

**审核规则现状**:
- ✅ 审核规则已配置化，保存在`audit_policy`表
- ✅ 提供管理界面给外部维护审核规则
- ✅ 支持规则CRUD、优先级调整、上线/下线等操作

**预审规则现状**:
- ❌ 预审规则硬编码在代码中（`RepayAuditSubmitService.java:77-108`）
- ❌ 无法通过配置动态调整预审规则
- ❌ 不支持AB实验的灵活配置
- ❌ 新增规则需要发布代码

### 1.2 需求目标

参考审核规则的设计，为预审规则提供完整的配置化方案：
1. ✅ 新建`pre_audit_policy`表存储预审规则
2. ✅ 提供CRUD接口给外部维护预审规则
3. ✅ 支持4个固定规则（不可删除、不可调整顺序）
4. ✅ 支持自定义可配置规则
5. ✅ 新增规则4：当天订单审核通过后二次发起无需审核

## 二、数据库设计

### 2.1 预审规则表（pre_audit_policy）

**表结构**:
```sql
CREATE TABLE repayfront.pre_audit_policy (
    id BIGINT(20) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键',
    rule_id VARCHAR(50) NOT NULL COMMENT '规则ID（唯一标识）',
    rule_name VARCHAR(100) NOT NULL COMMENT '规则名称',
    rule_type VARCHAR(20) NOT NULL COMMENT '规则类型：FIXED-固定规则，CUSTOM-可配置规则',
    priority INT NOT NULL COMMENT '优先级（数字越小优先级越高，固定规则优先级1-10，可配置规则优先级11-100）',
    rule_condition TEXT NOT NULL COMMENT '规则条件（JSON格式，存储规则特征配置）',
    status VARCHAR(20) NOT NULL DEFAULT 'ONLINE' COMMENT '规则状态：ONLINE-上线，OFFLINE-下线',
    description VARCHAR(500) NULL COMMENT '规则描述',
    created_by VARCHAR(150) NOT NULL DEFAULT 'SYS' COMMENT '创建人',
    updated_by VARCHAR(150) NOT NULL DEFAULT 'SYS' COMMENT '更新人',
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_rule_id (rule_id),
    INDEX idx_rule_type (rule_type),
    INDEX idx_priority (priority),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='预审规则表';
```

**设计说明**:
- ✅ 不包含rule_action字段（规则动作）
- ✅ 状态只有ONLINE和OFFLINE两种（不需要DESIGN状态）
- ✅ 创建时默认status=ONLINE
- ✅ 删除时更新status=OFFLINE（逻辑删除）

### 3.1 接口列表

| 接口名称 | 请求方式 | 路径 | 说明 | 优先级 |
|---------|---------|------|------|--------|
| 预审规则列表查询 | GET | /preAuditPolicy/list | 查询预审规则列表（支持分页、状态筛选，包含规则详情） | P0 |

### 4.3 Service层设计

**PreAuditPolicyService.java**:
```java
@Service
public class PreAuditPolicyService {

    @Autowired
    private IPreAuditPolicyRepository preAuditPolicyRepository;

    /**
     * 查询预审规则列表
     */
    public List<PreAuditPolicyPo> queryList(PreAuditRuleType ruleType, PreAuditRuleStatus status) {
        List<PreAuditPolicyPo> list = preAuditPolicyRepository.selectAll();

        // 过滤条件
        if (ruleType != null) {
            list = list.stream()
                .filter(item -> item.getRuleType() == ruleType)
                .collect(Collectors.toList());
        }
        if (status != null) {
            list = list.stream()
                .filter(item -> item.getStatus() == status)
                .collect(Collectors.toList());
        }

        return list;
    }

    /**
     * 新增预审规则（仅限可配置规则）
     */
    public void create(PreAuditPolicyPo po) {
        // 1. 校验ruleType必须为CUSTOM
        if (po.getRuleType() != PreAuditRuleType.CUSTOM) {
            throw new ClientException("只支持新增可配置规则");
        }

        // 2. 校验priority必须>=11
        if (po.getPriority() < 11) {
            throw new ClientException("可配置规则优先级必须>=11");
        }

        // 3. 生成ruleId
        po.setRuleId("RULE_" + System.currentTimeMillis());

        // 4. 设置默认状态为ONLINE
        po.setStatus(PreAuditRuleStatus.ONLINE);

        // 5. 插入数据库
        preAuditPolicyRepository.insert(po);
    }

    /**
     * 更新预审规则（仅限可配置规则）
     * 支持更新优先级实现优先级调整功能
     */
    public void update(PreAuditPolicyPo po) {
        // 1. 查询原规则
        Optional<PreAuditPolicyPo> optional = preAuditPolicyRepository.selectByRuleId(po.getRuleId());
        if (!optional.isPresent()) {
            throw new ClientException("规则不存在");
        }

        // 2. 校验规则类型
        PreAuditPolicyPo existing = optional.get();
        if (existing.getRuleType() == PreAuditRuleType.FIXED) {
            throw new ClientException("固定规则不允许修改");
        }

        // 3. 更新数据库
        preAuditPolicyRepository.update(po);
    }

    /**
     * 新增/更新预审规则（合并接口）
     */
    public void save(OperationType operationType, PreAuditPolicyPo po) {
        if (operationType == OperationType.CREATE) {
            create(po);
        } else if (operationType == OperationType.UPDATE) {
            update(po);
        }
    }

    /**
     * 删除预审规则（仅限可配置规则）
     * 实际是更新status=OFFLINE，不是物理删除
     */
    public void delete(String ruleId) {
        // 1. 查询原规则
        Optional<PreAuditPolicyPo> optional = preAuditPolicyRepository.selectByRuleId(ruleId);
        if (!optional.isPresent()) {
            throw new ClientException("规则不存在");
        }

        // 2. 校验规则类型
        PreAuditPolicyPo existing = optional.get();
        if (existing.getRuleType() == PreAuditRuleType.FIXED) {
            throw new ClientException("固定规则不允许删除");
        }

        // 3. ✅ 更新status为OFFLINE（逻辑删除）
        existing.setStatus(PreAuditRuleStatus.OFFLINE);
        preAuditPolicyRepository.update(existing);
    }

    /**
     * 加载预审规则（用于预审判断）
     */
    public List<PreAuditPolicyPo> loadPreAuditRules() {
        return preAuditPolicyRepository.selectOnline();
    }
}
```

```java
@RequestMapping("/preAuditPolicy")

    @Autowired

    }

        preAuditPolicyService.delete(ruleId);

    }
```

## 五、预审规则匹配逻辑

### 5.1 规则匹配引擎

**PreAuditRuleMatcher.java**:
```java
@Service
public class PreAuditRuleMatcher {

    @Autowired
    private PreAuditPolicyService preAuditPolicyService;

    /**
     * 匹配预审规则
     * 按优先级从高到低依次匹配，匹配到第一条规则后停止
     */
    public Optional<PreAuditPolicyPo> match(String uid, RepayAuditElementInfo info) {
        List<PreAuditPolicyPo> rules = preAuditPolicyService.loadPreAuditRules();

        // 按priority升序排序（数字越小优先级越高）
        rules.sort(Comparator.comparing(PreAuditPolicyPo::getPriority));

        // 依次匹配规则
        for (PreAuditPolicyPo rule : rules) {
            if (matchRule(uid, info, rule)) {
                return Optional.of(rule);
            }
        }

        return Optional.empty();
    }

    /**
     * 判断是否匹配规则
     */
    private boolean matchRule(String uid, RepayAuditElementInfo info, PreAuditPolicyPo rule) {
        // 解析ruleCondition（JSON）
        RuleCondition condition = JSON.parseObject(rule.getRuleCondition(), RuleCondition.class);

        // 根据规则类型匹配
        switch (rule.getRuleId()) {
            case "WHITELIST":
                return matchWhitelist(uid);
            case "SEAMLESS_V2":
                return matchSeamlessV2(info);
            case "ANY_REPAY":
                return matchAnyRepay(info);
            case "SAME_DAY_APPROVED":
                return matchSameDayApproved(uid, info.getRepayElementNo());
            default:
                return matchCustomRule(uid, info, condition);
        }
    }

    /**
     * 规则4：判断当天订单是否已审核通过
     */
    private boolean matchSameDayApproved(String uid, String repayElementNo) {
        Optional<AuditRecordPo> optional = auditRecordRepositoryProxy.selectOneUpToDate(uid, repayElementNo);
        if (!optional.isPresent()) {
            return false;
        }
        AuditRecordPo po = optional.get();

        // 审核状态为通过
        if (po.getAuditStatus() != AuditStatus.ACCEPTED) {
            return false;
        }

        // 审核完成时间为当天
        LocalDate localDate = LocalDate.now();
        LocalDate auditFinishDate = LocalDate.from(
            LocalDateTime.ofInstant(po.getAuditFinishTime().toInstant(), ZoneId.systemDefault())
        );

        return localDate.equals(auditFinishDate);
    }
}
```

### 5.2 预审主流程优化

**RepayAuditSubmitService.java** - 优化后：
```java
public RepayElementPreSubmitOutput.PreSubmitStatus preSubmitStatusByPreSubmit(
    String uid, RepayAuditElementInfo info, CommonRepayAuditConfig config) {

    // 1. 自动通过判断（白名单、审核开关等）
    if (autoPassByCommonAuditConfig(uid, info, config)) {
        return PreSubmitStatus.AUTO_PASS;
    }

    // 2. ✅ 从数据库加载预审规则并匹配
    Optional<PreAuditPolicyPo> matched = preAuditRuleMatcher.match(uid, info);
    if (matched.isPresent()) {
        // 记录预审结果到pre_audit_record表
        savePreAuditRecord(uid, info, matched.get());
        return matched.get().getRuleAction() == PreAuditRuleAction.AUTO_PASS
            ? PreSubmitStatus.AUTO_PASS
            : PreSubmitStatus.MANUAL_AUDIT;
    }

    // 3. 默认逻辑：需要审核
    return PreSubmitStatus.MANUAL_AUDIT;
}
```
