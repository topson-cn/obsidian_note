# 白名单失效机制改造说明

## 一、问题背景

### 1.1 现状分析

当前`/whiteList/update`接口实现的是**物理删除**逻辑：

```java
// RepayAuditServiceImpl.java:169-188
if (contained) {
    // 新增白名单
    bizMapRepository.insertOne(...);
} else {
    // ❌ 物理删除：直接从数据库删除记录
    bizMapRepository.deleteOneByKeyAndValue(bizMapCode, uid);
}
```

**问题点**:
1. ❌ 物理删除后无法追溯历史记录
2. ❌ 无法统计白名单失效情况
3. ❌ 无法支持AB实验数据分析
4. ❌ 不符合审计要求

### 1.2 需求目标

根据PRD需求（414291549），白名单需要支持：
1. ✅ 失效机制（状态管理）
2. ✅ 变更记录（新增/失效日志）
3. ✅ 失效原因追溯
4. ✅ 全流程数据追踪

### 1.3 实现方案

**采用方案**: 业务系统通过`/whiteList/update`接口主动失效白名单

**核心改动**: 将接口从**物理删除**改造为**逻辑删除**

**方案优势**:
- ✅ 实时性更好：业务系统可以实时失效白名单
- ✅ 实现更简单：无需开发定时任务
- ✅ 可靠性更高：直接依赖业务系统调用
- ✅ 追溯性更强：每次调用都记录操作人和原因

#### 2.2.3 Repository层改动

**BizMapDao.java** - 新增方法：
```java
public class BizMapDao {

    // ... 现有方法

    /**
     * 逻辑删除：更新白名单状态为EXPIRED
     */
    public int updateStatus(BizMapPo bizMapPo) {
        validator.validate(bizMapPo);

        return mapper.update(s -> s
            .set(status).equalTo(bizMapPo.getStatus().name()) // ✅ 更新状态
            .set(expiredAt).equalToWhenPresent(bizMapPo.getExpiredAt()) // ✅ 更新失效时间
            .set(expiredReason).equalToWhenPresent(bizMapPo.getExpiredReason()) // ✅ 更新失效原因
            .set(updatedBy).equalToWhenPresent(bizMapPo.getUpdatedBy())
            .where(key, isEqualTo(bizMapPo.getKeyCode().getMapKey()))
            .and(value, isEqualTo(bizMapPo.getValue())));
    }

    /**
     * 查询白名单（兼容历史数据：status为NULL也查询出来）
     * 注意：不在此处过滤status，而是在Service层判断
     */
    public Optional<BizMapPo> selectOneByKeyAndValue(BizMapPo bizMapPo) {
        validator.validate(bizMapPo);

        List<TBizMap> map = mapper.select(s -> s
            .where(key, isEqualTo(bizMapPo.getKeyCode().getMapKey()))
            .and(value, isEqualTo(bizMapPo.getValue()))); // ✅ 不过滤status，兼容历史数据

        if (CollectionUtils.isEmpty(map)) {
            return Optional.empty();
        }
        return Optional.of(modelMapper.map(map.get(0), BizMapPo.class));
    }
}
```

#### 2.2.4 Service层改动

**RepayAuditServiceImpl.java** - 核心改动：
```java
@Service
public class RepayAuditServiceImpl implements IRepayAuditService {

    @Autowired
    private IBizMapRepository bizMapRepository;

    @Autowired
    private IBizMapChangeLogRepository bizMapChangeLogRepository; // ✅ 新增

    @Override
    public void whiteListUpdate(String uid, boolean contained, String operator, String whiteType) {
        BizMapCode bizMapCode = BizMapCode.UID_WHITE_LIST;
        if(StringUtils.isNotBlank(whiteType)){
            bizMapCode = BizMapCode.valueOf(whiteType);
        }

        // ✅ 查询当前状态（兼容历史数据：status为NULL也视为ACTIVE）
        Optional<BizMapPo> currentOpt = bizMapRepository.selectOneByKeyAndValue(
            BizMapPo.builder().keyCode(bizMapCode).value(uid).build());
        boolean currentContained = currentOpt.isPresent() &&
            (currentOpt.get().getStatus() == null || currentOpt.get().getStatus() == BizMapStatus.ACTIVE);

        if (currentContained == contained) {
            return; // 状态未变化，无需处理
        }

        if (contained) {
            // ✅ 新增白名单，状态为ACTIVE
            bizMapRepository.insertOne(BizMapPo.builder()
                .keyCode(bizMapCode)
                .value(uid)
                .status(BizMapStatus.ACTIVE)
                .updatedBy(operator)
                .build());

            // ✅ 记录变更日志
            bizMapChangeLogRepository.insertOne(BizMapChangeLogPo.builder()
                .userId(uid)
                .changeType(BizMapChangeType.ADD)
                .changeReason("手动操作")
                .sourceSystem(SourceSystem.MANUAL)
                .operator(operator)
                .build());

        } else {
            // ✅ 逻辑删除：更新状态为EXPIRED
            bizMapRepository.updateStatus(BizMapPo.builder()
                .keyCode(bizMapCode)
                .value(uid)
                .status(BizMapStatus.EXPIRED)
                .expiredAt(new Date())
                .expiredReason("手动操作")
                .updatedBy(operator)
                .build());

            // ✅ 记录变更日志
            bizMapChangeLogRepository.insertOne(BizMapChangeLogPo.builder()
                .userId(uid)
                .changeType(BizMapChangeType.EXPIRE)
                .changeReason("手动操作")
                .sourceSystem(SourceSystem.MANUAL)
                .operator(operator)
                .build());
        }
    }

    private CommonRepayAuditConfig buildCommonRepayAuditConfig(String uid) {
        AuditPolicyMain policyMain = bizMapRepositoryProxy.selectAuditPolicyMain();

        // ✅ 查询白名单（兼容历史数据：status为NULL也视为ACTIVE）
        Optional<BizMapPo> optional = bizMapRepository.selectOneByKeyAndValue(
            BizMapPo.builder()
                .keyCode(BizMapCode.UID_WHITE_LIST)
                .value(uid)
                .build());

        // ✅ 兼容历史数据：status为NULL时，默认为ACTIVE
        boolean uidWhiteList = optional.isPresent() &&
            (optional.get().getStatus() == null || optional.get().getStatus() == BizMapStatus.ACTIVE);

        return CommonRepayAuditConfig.create(policyMain, uidWhiteList);
    }
}
```

## 三、改造对比

### 3.1 改造前后对比

| 维度 | 改造前 | 改造后 |
|------|--------|--------|
| **删除方式** | 物理删除 | 逻辑删除 |
| **状态管理** | ❌ 无状态 | ✅ ACTIVE/EXPIRED |
| **变更记录** | ❌ 无记录 | ✅ biz_map_change_log表 |
| **失效时间** | ❌ 无记录 | ✅ expired_at字段 |
| **失效原因** | ❌ 无记录 | ✅ expired_reason字段 |
| **数据追溯** | ❌ 无法追溯 | ✅ 完整追溯 |
| **预审判断** | 只判断是否存在 | 判断status=ACTIVE或NULL（兼容历史数据） |

### 3.2 接口调用示例

#### 改造前：
```bash
# 失效白名单（物理删除）
POST /repayAudit/whiteList/update
{
  "uid": "123456",
  "contained": false,
  "operator": "admin"
}
# 结果：数据库记录被删除，无法追溯
```

#### 改造后：

## 六、FAQ

### Q1: 为什么选择逻辑删除而不是物理删除？

**A**: 逻辑删除有以下优势：
1. ✅ 数据可追溯，符合审计要求
2. ✅ 支持数据分析和统计
3. ✅ 支持AB实验数据追踪
4. ✅ 避免误删除导致的数据丢失

### Q2: 现有数据如何处理？

**A**: 上线时需要执行数据迁移：
```sql
-- 将现有白名单状态设置为ACTIVE
UPDATE repayfront.biz_map
SET status = 'ACTIVE'
WHERE key = 'UID_WHITE_LIST' AND status IS NULL;
```

### Q3: 失效白名单是否会物理删除？

**A**: 不建议物理删除，原因：
1. 保留历史数据用于数据分析
2. 符合审计和合规要求
3. 避免误删除导致的数据丢失

### Q4: 业务系统如何调用接口失效白名单？

**A**: 业务系统只需调用现有的`/whiteList/update`接口：
```bash
POST /repayAudit/whiteList/update
{
  "uid": "123456",
  "contained": false,  # false表示失效
  "operator": "admin",
  "whiteType": "UID_WHITE_LIST"
}
```
