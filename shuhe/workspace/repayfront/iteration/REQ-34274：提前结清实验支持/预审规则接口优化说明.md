# 预审规则接口优化说明

> 更新时间: 2026-03-02
> 优化目标: 简化接口设计，提升开发效率

---

## 一、优化概述

本次对预审规则管理接口进行了优化，主要目的：
1. **简化接口数量**：减少冗余接口，降低维护成本
2. **提升开发效率**：合并相似接口，减少重复代码
3. **优化数据结构**：展开嵌套字段，提升可读性

---

## 二、接口改动详情

### 2.1 删除接口（2个）

| 接口名称 | 原路径 | 删除原因 |
|---------|--------|---------|
| 预审规则详情查询 | GET /preAuditPolicy/{ruleId} | 功能已在列表查询接口中实现 |
| 预审规则优先级调整 | POST /preAuditPolicy/priority | 功能已合并到更新接口 |

**删除原因说明**：
- **预审规则详情查询**：列表查询接口已包含完整的规则信息（包括展开的rule_condition），无需单独的详情接口
- **预审规则优先级调整**：通过更新接口的priority字段即可实现优先级调整，无需单独接口

### 2.2 合并接口（2个 → 1个）

| 原接口 | 新接口 | 合并方式 |
|--------|--------|---------|
| 预审规则新增<br>POST /preAuditPolicy/create | 预审规则新增/更新<br>POST /preAuditPolicy/save | 通过operation_type字段区分CREATE和UPDATE |
| 预审规则更新<br>POST /preAuditPolicy/update |  |  |

**合并优势**：
- ✅ 减少接口数量，降低维护成本
- ✅ 统一接口逻辑，减少重复代码
- ✅ 前端调用更简洁，只需一个接口
- ✅ 支持优先级调整功能（通过更新priority字段）

### 2.3 改造接口（1个）

| 接口名称 | 路径 | 改造内容 |
|---------|------|---------|
| 预审规则列表查询 | GET /preAuditPolicy/list | 展开rule_condition字段为平铺字段 |

**改造详情**：

**改造前**:
```json
{
  "ruleId": "RULE_001",
  "ruleName": "信用评分>600",
  "ruleCondition": "{\"featureKey\":\"creditScore\",\"operator\":\">\",\"tagValue\":\"600\"}"
}
```

**改造后**:
```json
{
  "ruleId": "RULE_001",
  "ruleName": "信用评分>600",
  "featureKey": "creditScore",
  "compareOp": ">",
  "tagValue": "600"
}
```

**改造优势**：
- ✅ 数据结构更清晰，可读性更强
- ✅ 前端无需解析JSON字符串
- ✅ 接口文档更易理解
- ✅ 减少前后端解析错误

## 四、核心接口设计

### 4.1 预审规则列表查询（改造）

**接口路径**: `GET /preAuditPolicy/list`

**请求参数**:
```java
{
  "ruleType": "CUSTOM",     // 可选：FIXED/CUSTOM
  "status": "ONLINE",       // 可选：ONLINE/OFFLINE
  "pageNum": 1,
  "pageSize": 10
}
```

**响应数据**:
```json
{
  "total": 100,
  "list": [
    {
      "ruleId": "RULE_001",
      "ruleName": "信用评分>600",
      "ruleType": "CUSTOM",
      "priority": 11,
      "status": "ONLINE",
      "description": "信用评分大于600的用户，无需审核",
      "featureKey": "creditScore",    // ✅ 展开字段
      "compareOp": ">",               // ✅ 比较运算符（Compare Operator）
      "tagValue": "600",              // ✅ 展开字段
      "createdAt": "2026-03-02 12:00:00",
      "updatedAt": "2026-03-02 12:00:00"
    }
  ]
}
```

**字段说明**:
- `featureKey`: 特征key（当前仅支持特征规则）
- `compareOp`: 比较运算符（Compare Operator），支持 >、<、=、>=、<=、!= 等
- `tagValue`: 标签值（具体的阈值或选项值）
- 注：每个可配置规则只支持单个特征条件

### 4.2 预审规则新增/更新（合并）

**接口路径**: `POST /preAuditPolicy/save`

**请求参数**:
```json
{
  "operationType": "CREATE",   // 必填：CREATE-新增，UPDATE-更新
  "rules": [                   // ✅ 支持批量操作
    {
      "ruleId": "RULE_001",    // 更新时必填，新增时忽略
      "ruleName": "信用评分>600",
      "priority": 11,
      "featureKey": "creditScore",
      "compareOp": ">",        // ✅ 比较运算符（Compare Operator）
      "tagValue": "600",
      "description": "信用评分大于600的用户，无需审核"
    },
    {
      "ruleId": "RULE_002",
      "ruleName": "订单金额<10000",
      "priority": 12,
      "featureKey": "orderAmount",
      "compareOp": "<",
      "tagValue": "10000",
      "description": "订单金额小于10000的用户，无需审核"
    }
  ]
}
```

**响应数据**:
```json
{
  "results": [                 // ✅ 返回每条规则的处理结果
    {
      "ruleId": "RULE_001",
      "success": true,
      "message": "操作成功"
    },
    {
      "ruleId": "RULE_002",
      "success": true,
      "message": "操作成功"
    }
  ]
}
```

**操作类型说明**:

| operationType | 说明 | ruleId要求 | 其他要求 |
|--------------|------|-----------|---------|
| CREATE | 新增规则 | 忽略（系统自动生成） | priority必须>=11，支持批量新增 |
| UPDATE | 更新规则 | 必填 | 固定规则（FIXED）不允许更新，支持批量更新 |

**支持功能**:
- ✅ 新增可配置规则（支持批量）
- ✅ 更新可配置规则（包括优先级，支持批量）
- ✅ 优先级调整（通过更新priority字段）
- ✅ 批量操作处理（返回每条规则的处理结果）

### 4.3 预审规则删除（保持不变）

**接口路径**: `DELETE /preAuditPolicy/{ruleId}`

**校验逻辑**:
- 固定规则（FIXED）不允许删除
- 只有可配置规则（CUSTOM）允许删除
- 删除时实际更新status=OFFLINE（逻辑删除）

## 五、代码实现要点

### 5.1 列表查询接口改造

```java
/**
 * 构建响应对象（展开rule_condition字段）
 */
private PreAuditPolicyVo buildVo(PreAuditPolicyPo po) {
    PreAuditPolicyVo vo = new PreAuditPolicyVo();
    vo.setRuleId(po.getRuleId());
    vo.setRuleName(po.getRuleName());
    vo.setRuleType(po.getRuleType());
    vo.setPriority(po.getPriority());
    vo.setStatus(po.getStatus());
    vo.setDescription(po.getDescription());
    vo.setCreatedAt(po.getCreatedAt());
    vo.setUpdatedAt(po.getUpdatedAt());

    // ✅ 解析rule_condition JSON并展开
    RuleCondition condition = JSON.parseObject(po.getRuleCondition(), RuleCondition.class);
    vo.setFeatureKey(condition.getFeatureKey());
    vo.setCompareOp(condition.getCompareOp());    // ✅ 比较运算符
    vo.setTagValue(condition.getTagValue());

    return vo;
}
```

### 5.2 新增/更新接口实现（支持批量）

```java
/**
 * 新增/更新预审规则（合并接口，支持批量操作）
 */
@PostMapping("/save")
public PreAuditPolicySaveResp save(@RequestBody @Valid PreAuditPolicySaveReq req) {
    List<PreAuditPolicySaveResult> results = new ArrayList<>();

    // ✅ 循环处理每条规则
    for (PreAuditPolicySaveReq.RuleSaveItem item : req.getRules()) {
        try {
            PreAuditPolicyPo po = buildPo(item);

            // 根据operation_type判断新增还是更新
            if (req.getOperationType() == OperationType.CREATE) {
                preAuditPolicyService.create(po);
            } else if (req.getOperationType() == OperationType.UPDATE) {
                preAuditPolicyService.update(po);
            }

            results.add(PreAuditPolicySaveResult.builder()
                .ruleId(po.getRuleId())
                .success(true)
                .message("操作成功")
                .build());
        } catch (Exception e) {
            // ✅ 单条规则失败不影响其他规则
            results.add(PreAuditPolicySaveResult.builder()
                .ruleId(item.getRuleId())
                .success(false)
                .message(e.getMessage())
                .build());
        }
    }

    return PreAuditPolicySaveResp.builder().results(results).build();
}
```

## 六、前端对接指南

### 6.1 列表查询（前端改动）

**改动前**:
```javascript
// 需要解析JSON字符串
const condition = JSON.parse(rule.ruleCondition);
console.log(condition.featureKey, condition.operator, condition.tagValue);
```

**改动后**:
```javascript
// 直接使用展开的字段
console.log(rule.featureKey, rule.compareOp, rule.tagValue);
```

### 6.2 新增/更新（前端改动，支持批量）

**改动前**:
```javascript
// 需要调用两个不同的接口
if (isCreate) {
  await axios.post('/preAuditPolicy/create', createData);
} else {
  await axios.post('/preAuditPolicy/update', updateData);
}
```

**改动后**:
```javascript
// ✅ 只需调用一个接口，支持批量操作
const result = await axios.post('/preAuditPolicy/save', {
  operationType: 'CREATE',
  rules: [
    {
      ruleName: '信用评分>600',
      priority: 11,
      featureKey: 'creditScore',
      compareOp: '>',
      tagValue: '600',
      description: '信用评分大于600的用户，无需审核'
    },
    {
      ruleName: '订单金额<10000',
      priority: 12,
      featureKey: 'orderAmount',
      compareOp: '<',
      tagValue: '10000',
      description: '订单金额小于10000的用户，无需审核'
    }
  ]
});

// ✅ 查看每条规则的处理结果
result.data.results.forEach(item => {
  console.log(`${item.ruleId}: ${item.success ? '成功' : '失败'} - ${item.message}`);
});
```

### 6.3 优先级调整（前端改动）

**改动前**:
```javascript
// 需要调用专门的优先级调整接口
await axios.post('/preAuditPolicy/priority', {
  rulePriorities: [
    {ruleId: 'RULE_001', priority: 11},
    {ruleId: 'RULE_002', priority: 12}
  ]
});
```

**改动后**:
```javascript
// ✅ 通过更新接口批量调整优先级
await axios.post('/preAuditPolicy/save', {
  operationType: 'UPDATE',
  rules: [
    {ruleId: 'RULE_001', priority: 11},  // ✅ 直接更新priority字段
    {ruleId: 'RULE_002', priority: 12}
  ]
});
```
