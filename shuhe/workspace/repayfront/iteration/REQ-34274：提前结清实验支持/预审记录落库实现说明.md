# 预审记录落库实现说明

## 一、需求背景

### 1.1 现状分析

**当前实现**（`RepayAuditServiceImpl.java:88-97`）:
```java
@Override
public List<RepayElementPreSubmitOutput> repayAuditPreSubmit(String uid, String bizSerial, List<RepayAuditElementInfo> repayAuditElementInfoList) {
    CommonRepayAuditConfig config = buildCommonRepayAuditConfig(uid);
    return repayAuditElementInfoList.stream()
        .map(info -> RepayElementPreSubmitOutput.builder()
            .repayCategory(info.getRepayCategory())
            .repayElementNo(info.getRepayElementNo())
            .status(repayAuditSubmitService.preSubmitStatusByPreSubmit(uid, info, config))
            .build())
        .collect(Collectors.toList());
}
```

**问题点**:
- ❌ 预审结果未落库，无法追溯
- ❌ 无法统计预审通过率
- ❌ 无法支持AB实验数据分析
- ❌ 无法追踪预审规则匹配情况

### 1.2 需求目标

根据PRD需求，需要：
1. ✅ 新建`pre_audit_record`表存储预审记录
2. ✅ 每次预审判断后将结果落库
3. ✅ 记录flowId、userId、orderId、auditResult、matchRuleId、matchType
4. ✅ 支持AB实验数据追踪和分析

## 二、数据库设计

### 2.1 预审记录表（pre_audit_record）

**表结构**:
```sql
CREATE TABLE repayfront.pre_audit_record (
    id BIGINT(20) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键',
    flow_id VARCHAR(50) NOT NULL COMMENT '业务流水号',
    user_id VARCHAR(50) NOT NULL COMMENT '用户ID',
    order_id VARCHAR(50) NOT NULL COMMENT '订单ID',
    audit_result VARCHAR(20) NOT NULL COMMENT '预审结果：NEED_AUDIT-需要审核，NO_NEED-无需审核',
    match_rule_id VARCHAR(50) NULL COMMENT '匹配的规则ID（如WHITELIST、SEAMLESS_V2、RULE_001等）',
    match_type VARCHAR(20) NOT NULL COMMENT '匹配类型：WHITELIST-白名单，RULE-规则，DEFAULT-默认',
    created_by VARCHAR(150) NOT NULL DEFAULT 'SYS' COMMENT '创建人',
    updated_by VARCHAR(150) NOT NULL DEFAULT 'SYS' COMMENT '更新人',
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_flow_id_order (flow_id, order_id),  -- 同一个flow_id+order_id只有一条记录
    INDEX idx_user_order (user_id, order_id),
    INDEX idx_audit_result (audit_result),
    INDEX idx_match_rule_id (match_rule_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='预审记录表';
```

### 2.2 字段说明

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| id | BIGINT | 主键 | 1234567890 |
| flow_id | VARCHAR(50) | 业务流水号 | TQJQ_20260228120000_abc123 |
| user_id | VARCHAR(50) | 用户ID | 123456 |
| order_id | VARCHAR(50) | 订单ID（还款元素号） | order_001 |
| audit_result | VARCHAR(20) | 预审结果 | NEED_AUDIT 或 NO_NEED |
| match_rule_id | VARCHAR(50) | 匹配的规则ID | WHITELIST、SEAMLESS_V2、RULE_001等 |
| match_type | VARCHAR(20) | 匹配类型 | WHITELIST、RULE、DEFAULT |
| created_at | TIMESTAMP | 创建时间 | 2026-02-28 12:00:00 |

### 2.3 枚举值说明

**audit_result（预审结果）**:
- `NEED_AUDIT` - 需要审核
- `NO_NEED` - 无需审核

**match_rule_id（匹配的规则ID）**:
- `WHITELIST` - 白名单匹配
- `SEAMLESS_V2` - 轻资产订单
- `ANY_REPAY` - 随借随还订单
- `SAME_DAY_APPROVED` - 当天订单审核通过后二次发起
- `RULE_001` - 自定义规则1
- `RULE_002` - 自定义规则2
- `null` - 默认逻辑（未匹配到任何规则）

**match_type（匹配类型）**:
- `WHITELIST` - 白名单类型
- `RULE` - 规则类型（固定规则或可配置规则）
- `DEFAULT` - 默认逻辑（未匹配到任何规则）

### 3.3 Service层改造

**RepayAuditServiceImpl.java** - 改造后：
```java
@Service
public class RepayAuditServiceImpl implements IRepayAuditService {

    @Autowired
    private PreAuditRecordRepository preAuditRecordRepository; // ✅ 新增

    @Override
    public List<RepayElementPreSubmitOutput> repayAuditPreSubmit(
        String uid, String flowId, List<RepayAuditElementInfo> repayAuditElementInfoList) {

        CommonRepayAuditConfig config = buildCommonRepayAuditConfig(uid);

        return repayAuditElementInfoList.stream()
            .map(info -> {
                // 1. 预审判断
                PreSubmitStatus status = repayAuditSubmitService.preSubmitStatusByPreSubmit(
                    uid, info, config);

                // 2. ✅ 保存预审记录到pre_audit_record表
                PreAuditRecordPo record = PreAuditRecordPo.builder()
                    .flowId(flowId)
                    .userId(uid)
                    .orderId(info.getRepayElementNo())
                    .auditResult(status == PreSubmitStatus.AUTO_PASS
                        ? PreAuditResult.NO_NEED
                        : PreAuditResult.NEED_AUDIT)
                    .matchRuleId(determineMatchRuleId(uid, info, status)) // 确定匹配的规则ID
                    .matchType(determineMatchType(uid, info, status)) // 确定匹配类型
                    .build();

                preAuditRecordRepository.insert(record);

                // 3. 返回结果
                return RepayElementPreSubmitOutput.builder()
                    .repayCategory(info.getRepayCategory())
                    .repayElementNo(info.getRepayElementNo())
                    .status(status)
                    .build();
            })
            .collect(Collectors.toList());
    }

    /**
     * ✅ 确定匹配的规则ID
     */
    private String determineMatchRuleId(String uid, RepayAuditElementInfo info, PreSubmitStatus status) {
        // 如果是自动通过，需要确定匹配的规则ID
        if (status == PreSubmitStatus.AUTO_PASS) {
            // 1. 检查白名单
            if (isWhitelistActive(uid)) {
                return "WHITELIST";
            }

            // 2. 检查轻资产订单
            OrdersInfo ordersInfo = loanSafetyRpc.getOrderInfoWithoutPlan(uid, info.getRepayElementNo());
            if (fundConfigGather.isSeamlessV2(ordersInfo.getBank())) {
                return "SEAMLESS_V2";
            }

            // 3. 检查随借随还订单
            if (ProductEnum.ANY_REPAY.name().equals(ordersInfo.getProduct())) {
                return "ANY_REPAY";
            }

            // 4. 检查当天订单审核通过后二次发起
            if (isSameDayApproved(uid, info.getRepayElementNo())) {
                return "SAME_DAY_APPROVED";
            }

            // 5. 检查可配置规则
            Optional<PreAuditPolicyPo> matched = preAuditRuleMatcher.match(uid, info);
            if (matched.isPresent()) {
                return matched.get().getRuleId();
            }
        }

        // 默认逻辑：返回null
        return null;
    }

    /**
     * ✅ 确定匹配类型
     */
    private PreAuditMatchType determineMatchType(String uid, RepayAuditElementInfo info, PreSubmitStatus status) {
        String matchRuleId = determineMatchRuleId(uid, info, status);

        if (matchRuleId == null) {
            return PreAuditMatchType.DEFAULT;
        } else if ("WHITELIST".equals(matchRuleId)) {
            return PreAuditMatchType.WHITELIST;
        } else {
            return PreAuditMatchType.RULE;
        }
    }

    /**
     * 判断当天订单是否已审核通过
     */
    private boolean isSameDayApproved(String uid, String repayElementNo) {
        Optional<AuditRecordPo> optional = auditRecordRepositoryProxy.selectOneUpToDate(uid, repayElementNo);
        if (!optional.isPresent()) {
            return false;
        }
        AuditRecordPo po = optional.get();

        // 审核状态为通过
        if (po.getAuditStatus() != AuditStatus.ACCEPTED) {
            return false;
        }

        // 审核完成时间为当天
        LocalDate localDate = LocalDate.now();
        LocalDate auditFinishDate = LocalDate.from(
            LocalDateTime.ofInstant(po.getAuditFinishTime().toInstant(), ZoneId.systemDefault())
        );

        return localDate.equals(auditFinishDate);
    }
}
```

### 3.4 Controller层改造

**RepayAuditController.java** - 改造后：
```java
@RestController
@RequestMapping("/repayAudit")
public class RepayAuditController {

    @Autowired
    private IRepayAuditService repayAuditService;

    @PostMapping("/preSubmit")
    @ApiOperation("还款审核预提交")
    public RepayAuditPreSubmitResp repayAuditPreSubmit(@RequestBody @Valid RepayAuditPreSubmitReq req) {

        String flowId = req.getBizSerial(); // ✅ 从请求参数获取flowId

        // 1. ✅ 账务校验（在预审之前）
        RepayCheckResult checkResult = repayCheckService.check(req.getUid(), req.getRepayElementNo());
        if (!checkResult.isSuccess()) {
            // 记录账务校验埋点
            saveRepayCheckLog(flowId, req.getUid(), req.getRepayElementNo(), checkResult);
            // 返回校验失败
            return buildFailResp(checkResult.getFailReason());
        }
        // 记录账务校验成功埋点
        saveRepayCheckLog(flowId, req.getUid(), req.getRepayElementNo(), checkResult);

        // 2. 预审判断（包含✅预审记录落库）
        List<RepayAuditElementInfo> repayAuditElementInfoList = req.getRepayElementPreSubmitReqList().stream()
            .map(item -> modelMapper.map(item, RepayAuditElementInfo.class))
            .collect(Collectors.toList());

        List<RepayElementPreSubmitOutput> outputList = repayAuditService.repayAuditPreSubmit(
            req.getUid(),
            flowId, // ✅ 传递flowId
            repayAuditElementInfoList
        );

        // 3. ✅ 注意：预审记录已在repayAuditPreSubmit方法中保存到pre_audit_record表

        return buildSuccessResp(outputList);
    }
}
```

## 四、数据流转图

```
用户点击提前结清按钮
    ↓
前端生成flowId
    ↓
前端：调用/repayAudit/preSubmit接口
    ↓
后端：RepayAuditController.repayAuditPreSubmit()
    ↓
后端：账务校验（✅新增）
    ├─ 成功：继续
    └─ 失败：返回错误
    ↓
后端：预审判断（RepayAuditSubmitService.preSubmitStatusByPreSubmit）
    ↓
后端：✅ 保存预审记录到pre_audit_record表
    ├─ flow_id: TQJQ_20260228120000_abc123
    ├─ user_id: 123456
    ├─ order_id: order_001
    ├─ audit_result: NO_NEED（或NEED_AUDIT）
    ├─ match_rule_id: WHITELIST（或SEAMLESS_V2、RULE_001等）
    └─ match_type: WHITELIST（或RULE、DEFAULT）
    ↓
后端：返回预审结果给前端
```
