## 功能说明

决策节点，根据用户特征、订单信息、渠道可用性等多维度因素，智能选择最优的还款渠道。

### 决策目标
1. **提升成功率**：选择成功率最高的支付渠道
2. **降低成本**：优先选择费率低的渠道
3. **优化体验**：考虑用户偏好和历史行为
4. **风险控制**：识别异常交易，选择合适渠道

## 决策维度

### 1. 用户维度
- **用户等级**：普通用户/会员用户/VIP用户
- **信用评分**：基于历史还款行为
- **偏好渠道**：用户历史使用的支付方式
- **设备类型**：iOS/Android/Web

### 2. 订单维度
- **还款金额**：不同金额区间选择不同渠道
- **订单类型**：普通订单/分期订单/提前结清
- **逾期天数**：是否逾期，逾期多久
- **资方类型**：不同资方的渠道偏好

### 3. 渠道维度
- **渠道可用性**：实时监控渠道是否可用
- **渠道成功率**：近期成功率统计
- **渠道费率**：不同渠道的手续费
- **渠道限额**：单笔/单日限额

### 4. 风控维度
- **风险等级**：高风险用户渠道限制
- **异常交易检测**：频繁还款、大额还款等
- **黑名单检测**：用户/设备/IP黑名单

## 决策规则（示例）

### 规则1：大额还款渠道选择
```
IF 还款金额 > 50000 AND 用户信用评分 > 80:
    渠道优先级 = [银行卡代扣, 支付宝大额, 微信支付]
ELSE IF 还款金额 > 50000 AND 用户信用评分 <= 80:
    渠道优先级 = [银行卡代扣]  # 高额低信用只允许银行卡
```

### 规则2：会员用户优先级
```
IF 用户类型 == "VIP会员":
    渠道费率折扣 = 0.8  # 8折手续费
    渠道优先级 = [用户偏好渠道, 支付宝, 微信支付, 银行卡]
```

### 规则3：逾期订单处理
```
IF 逾期天数 > 30:
    强制渠��� = [银行卡代扣]  # 逾期长的订单强制使用银行卡
    禁用渠道 = [支付宝SDK, 微信支付]  # 禁用用户主动支付
```

### 规则4：渠道降级策略
```
IF 首选渠道.成功率 < 0.9:
    执行渠道降级
    选择备选渠道列表中成功率最高的渠道
```

### 规则5：特殊资方处理
```
IF 资方 == "AJTRUST_ZSPH_5":
    # 已结束，跳过两方拆扣逻辑
    使用标准拆扣逻辑
ELSE IF 资方需要两方拆扣:
    拆分为资方部分和平台部分
    分别选择渠道
```
