## 业务流程概述

轻资产同步还款提交流程是轻资产还款场景的统一主流程，采用有状态模式运行，将同步提交和异步结果处理整合在同一条业务流中。流程同步阶段完成参数校验、还款申请保存、分期处理、扣款单拆分、数据持久化、分期锁定及扣款发起，并组织返回报文同步响应调用方；异步阶段处理扣款结果确认、入账、额度同步、解锁及消息通知等后续操作。

### 核心功能
1. **准入校验**：验证还款金额合法性、检查重复还款请求
2. **申请保存**：保存还款申请及关联信息
3. **轻资产还款单处理**：初始化分期信息、拆分还款单、还款试算、拆分扣款单、持久化
4. **资源锁定与扣款**：锁定分期、填充同步上下文、发起扣款、组织返回报文
5. **异步结果处理**：扣款结果确认、话术策略（条件分支）、入账、额度同步、解锁、消息通知

## 异常处理策略

### 全局异常策略
- **重试类型**: normal
- **重试次数**: 5次
- **重试间隔**: 60秒
- **失败后状态**: ERROR (错误)

### 节点级异常策略

| 节点 | 处理器 | 异常策略 |
|------|--------|----------|
| 准入校验~同步扣款执行 (阶段2~8) | P010002 ~ P026000 | 默认（跟随全局策略） |
| 扣款结果确认事件 | P070030 | 重试1次，间隔60秒，失败后暂停(PAUSED) |
| 还款入账事件 | PL070038 | 重试1次，间隔60秒，失败后暂停(PAUSED) |
| 轻资产额度同步 | PL070085 | 重试1次，间隔60秒，失败后暂停(PAUSED) |
| 还款入账后置事件 | P070999 | 重试1次，间隔60秒，失败后暂停(PAUSED) |
| 等待扣款结果 | PL070029 | 重试1次，间隔60秒，失败后暂停(PAUSED) |
| 还款交易解锁 | PL070070 | 重试1次，间隔60秒，失败后暂停(PAUSED) |
| 发送结果消息 | PL070080 | 重试1次，间隔60秒，失败后暂停(PAUSED) |

> **说明**: 异步阶段节点（P070030之后）采用"重试1次 + 失败暂停"策略，避免无限重试，支持人工介入排查。
