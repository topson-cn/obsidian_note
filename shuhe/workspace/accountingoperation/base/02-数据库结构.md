## 数据库设计特点

### 1. 多数据库架构优势
- **读写分离** - 主业务库支持读写，数仓只读查询
- **业务隔离** - 不同业务域使用不同表
- **性能优化** - 报表查询不影响主业务
- **数据集成** - DataHub 提供数据交换能力

### 2. 表命名规范
- 主表使用全名小写下划线分隔：`account_adjust_trans_log`
- 关联表使用前缀标识：`offline_repay_*`, `manual_deduct_*`
- 工单表统一后缀：`*_work_order`
- 交易日志表统一后缀：`*_trans_log`
- 配置表使用前缀：`adjust_*`, `t_*`

### 3. 通用字段
大多数业务表包含以下通用字段：
- `id` - 主键
- `create_time` / `created_at` - 创建时间
- `update_time` / `updated_at` - 更新时间
- `create_by` / `created_by` - 创建人
- `update_by` / `updated_by` - 更新人
- `status` - 状态

### 4. 索引策略
- 主键索引：所有表的 `id` 字段
- 业务索引：订单号、工单号、用户 ID 等
- 时间索引：创建时间、更新时间
- 组合索引：根据查询频率设计

## 事务管理

### 事务管理器配置
```xml
<bean id="transactionManager0"
      class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="dataSource0"/>
</bean>
```

### 事务注解使用
- `@Transactional` - 声明式事务
- 支持多事务管理器配置
- 分布式事务通过业务流程保证

## 数据库访问模式

### 1. Repository 模式
复杂业务使用自定义 Repository：
- RecreateOrderSnapshotRepository
- 其他复杂查询仓储

### 2. Query Service 模式
查询服务分离：
- BankFlowQueryService
- OverFlowPaymentQueryService (新增)
- 其他专用查询服务

### 3. Batch Query 优化
批量查询避免 N+1 问题：
- 批量查询溢缴款
- 批量查询银行流水
- 批量查询订单信息
